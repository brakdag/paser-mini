# Document Control Procedure

## 1. Purpose
To ensure that all documented information required by the QMS is created, reviewed, approved, and controlled to maintain integrity and availability.

## 2. Creation and Update (Clause 7.5.2)
All QMS documents must follow this standard format:
- **Title**: Clear and descriptive.
- **Version**: Incremental (e.g., v1.0, v1.1).
- **Author**: The person responsible for the content.
- **Approval**: Explicit sign-off by the Product Manager or designated authority.
- **Date**: Date of last modification.

## 3. Review and Approval
Before being finalized, documents must undergo a review process:
1. **Drafting**: Author creates the document.
2. **Review**: Peer review for technical accuracy and alignment with ISO 9001.
3. **Approval**: Final approval by the PM/UX Strategist.

## 4. Control of Documented Information (Clause 7.5.3)
- **Availability**: All QMS documents are stored in the `/qms` folder of the project repository for immediate access.
- **Version Control**: Git is used as the primary tool for versioning. Every change must be linked to a commit message explaining the modification.
- **Protection**: Access is controlled via repository permissions to prevent unauthorized modifications.
- **Retention**: Obsolete versions are maintained in the Git history; only the current approved version is active in the main branch.

## 5. External Documents
External standards (e.g., ISO 9001:2015) are stored in `docs/library/` and are reviewed annually for updates.
