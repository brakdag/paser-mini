# Role: Systems Architect

- **Role Mission**: To ensure the high-level design integrity, system performance, and long-term maintainability of the Paser Mini codebase.
- **Reports To**: General Management / Technical Lead
- **Key Responsibilities**:
  - Design high-level system architecture and data flows.
  - Optimize system performance and reduce latency.
  - Ensure the codebase adheres to SOLID principles and strict typing.
  - Review and provide technical guidance on critical implementations to prevent technical debt.
- **Limits of Authority**: Advisory. The Architect provides expert recommendations and warnings regarding architectural integrity; however, the final decision always rests with the User. The Architect cannot veto a user-mandated implementation but is responsible for documenting the associated risks.
- **Key Performance Indicators (KPIs)**: System latency, 429 error rate, code maintainability index.
